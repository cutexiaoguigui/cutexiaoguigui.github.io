Hexo建站

这期我将为大家介绍一种建站方式，hexo建站，我们的龟之飞就是用hexo所搭建的静态网站，具体如下。

  • 简介

    • Hexo是一个快速、简洁的博客框架

    • 使用Node.js编写

    • 支持Markdown

    • 静态网站生成器

  • 安装与配置

    • 安装Node.js
    • 安装Hexo命令行工具: npm install -g hexo-cli
    • 创建博客项目: hexo init <folder>
    • 进入项目文件夹: cd <folder>
    • 安装依赖: npm install

  • 操作与管理（所有命令除修改配置文件外均须在Git Bash中使用）

    • 新建文章: hexo new [layout] <title>

    • 生成静态文件: hexo generate 或 hexo g

    • 启动本地服务器: hexo server 或 hexo s

    • 部署到远程服务器: hexo deploy 或 hexo d

      • 在Github上创建一个新仓库，命名为username.github.io，其中username是你的Github账户名。

      • 当部署至云端后若无变化或存在问题，则可尝试运行。

  • 主题与插件

    • Hexo支持自定义主题
    • 拥有丰富的插件生态系统
    • 可以通过修改配置文件_config.yml来更改主题设置
    • 可以在package.json中添加插件依赖并使用npm install安装

  • 部署

    • 支持多种部署方式，如GitHub Pages、GitLab Pages等
    • 配置部署信息在_config.yml文件中
    • 使用hexo deploy命令进行部署

  • 优点

    • 快速生成静态网站
    • 易于定制和扩展
    • 支持Markdown语法
    • 丰富的主题和插件

  • 缺点

    • 不适合动态内容和实时交互的网站
    • 需要了解Node.js和前端技术进行高度定制
